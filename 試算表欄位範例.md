# Google 試算表資料欄位範例

以下為 **商品** 工作表與 **設定** 工作表的欄位名稱與範例。**第 1 列（最上面一列）必須是標題列**（規格1、規格2、商品主圖等），第 2 列起才是商品資料；若規格1～規格4 寫在第 2 列以後，系統會讀不到，顧客頁就只會顯示一個規格。

---

## 一、商品工作表（分頁名稱可為「商品」或使用第一個分頁）

| 欄位名稱 | 說明 | 必填 | 範例 |
|----------|------|------|------|

| **推薦** | 推薦標籤，填 Y 或 1 表示是 | 選填 | |
| **新品** | 新品標籤，填 Y 或 1 表示是 | 選填 | Y |
| **新上架** | 新上架標籤，填 Y 或 1 表示是 | 選填 | Y |
| **貨況** | 貨況（顧客頁顯示用；填 現貨 或 預購，後台新增時下拉選單會寫入此欄） | 選填 | 現貨 或 預購 |
| **狀態** | 上架／下架 | 選填 | 上架 |
| **描述** | 描述／說明 | 選填 | |

### 第一列標題列範例（可複製到試算表第一列）

後台可輸入**日幣價格**，依「設定」工作表的**匯率**自動帶出**台幣售價**；**利潤**＝台幣售價－成本（自動計算）。試算表標題可依下列順序（順序可自訂，名稱對應即可）：

```
商品名稱	分類	子分類	規格	售價(JPY)	商品主圖	規格圖片	商品介紹	上架日期	售價	成本	利潤	熱銷	推薦	新品	新上架	貨況	狀態	描述	庫存	規格庫存	規格售價
```
（**售價(JPY)** 為選填，不需要可刪除該欄。）

### 第二列資料列範例（一筆商品）

```
三麗鷗星星月筆筒	居家生活	抱枕	酷洛米, 凱蒂貓, 大耳狗	1200	https://example.com/main.jpg	https://example.com/kuromi.jpg, https://example.com/kitty.jpg, https://example.com/cinnamoroll.jpg	日本限量限定	2026-03-03	350	280	70	Y		Y	Y	現貨	上架	限量款抱枕	10	2,3,5
```

說明：後台輸入**日幣** 1200、**設定**工作表匯率 0.23 時，會自動帶出**台幣售價** 276（或 350 可手動改）；**成本** 280、**利潤** 由後台自動計算（售價－成本）後儲存寫入試算表。

## 二、設定工作表（分頁名稱請設為「設定」）

**後台商品管理**：匯率可直接在「商品管理」頁上方輸入，會存於本機瀏覽器，不一定要在試算表「設定」工作表設定。

|----------|------|------|
| **匯率** 或 **rate** | 1 日圓 = 多少台幣（數字）；後台未設定時可由此讀取 | 0.23 |

## 四、紅利紀錄工作表（分頁名稱請設為「紅利紀錄」）

後台「紅利點數」按 **同步至試算表** 時會寫入此分頁。  
規則：**消費滿 NT$100 集 1 點**、**1 點折 NT$1**、**365 天內有效**，到期自動失效。

| 欄位名稱 | 說明 | 必填 | 範例 |
|----------|------|------|------|
| **紀錄ID** | 每筆異動唯一 ID | 必填 | PTME1234ABCD |
| **日期** | 異動日期 YYYY-MM-DD | 必填 | 2026-06-04 |
| **電話** | 顧客電話 | 選填 | 0912345678 |
| **Line ID** | 顧客 Line ID（電話、Line 至少填一項） | 選填 | @maaru_customer |
| **姓名** | 顧客姓名 | 選填 | 王小明 |
| **類型** | 發放／折抵／失效／調整 | 必填 | 發放 |
| **點數** | 正數=增加、負數=扣除 | 必填 | 12 |
| **剩餘** | **僅「發放」「調整」**時記錄尚未用完的點數；折抵後會從舊批次扣減 | 選填 | 12 |
| **到期日** | **發放／調整**的到期日（發放日 +365 天） | 選填 | 2027-06-04 |
| **訂單編號** | 關聯訂單（手動調整可留空） | 選填 | ORD00001 |
| **備註** | 說明 | 選填 | 消費滿 100 元集點 |

### 第一列標題列範例（可整列複製）

```
紀錄ID	日期	電話	Line ID	姓名	類型	點數	剩餘	到期日	訂單編號	備註
```

### 資料列範例（同一顧客三種異動）

```
PT001	2026-06-04	0912345678	@maaru_customer	王小明	發放	12	12	2027-06-04	ORD00001	消費滿 100 元集點
PT002	2026-06-10	0912345678	@maaru_customer	王小明	折抵	-5	0		ORD00002	訂單折抵
PT003	2027-06-05	0912345678	@maaru_customer	王小明	失效	-7	0			到期自動失效（原發放 PT001）
```

### 類型說明

| 類型 | 說明 |
|------|------|
| **發放** | 訂單完成集點，`剩餘`＝可再用點數，`到期日`＝發放日+365天 |
| **折抵** | 訂單使用紅利，`點數` 為負數 |
| **失效** | 到期未用完，系統或手動執行「到期失效檢查」產生 |
| **調整** | 後台手動加點或扣點 |

## 五、欄位對照速查（標題可用的名稱）

- **商品主圖**：商品主圖、圖片、圖片URL、image、imageUrl  
- **規格**：規格、顏色、option、variant（多個用逗號分隔，例：凱蒂貓,美樂,布丁狗）  
- **規格1～規格4**：分欄時用，各欄填一個規格名稱，API 會合併後在顧客頁顯示多個選項  
- **規格圖片**：規格圖片、variantImages  
- **商品介紹**：商品介紹、介紹、introduction  
- **上架日期**：上架日期、上架時間、publishedAt  
- **熱銷**：熱銷、hot（Y／1 表示是）  
- **推薦**：推薦、recommended  
- **新品**：新品、isNew
- **新上架**：新上架、isNewListing
- **貨況**：貨況、現貨預購、現貨/預購、stockType（填「現貨」或「預購」，顧客頁會顯示；後台新增商品時下拉選現貨/預購會寫入此欄）

**訂單**

- **訂單編號**：訂單編號、id  
- **待結清總金額**：待結清總金額、總計、total

- **使用紅利**：使用紅利、pointsUsed  
- **獲得紅利**：獲得紅利、pointsEarned

**紅利紀錄**

## 六、規格欄怎麼填才會在顧客頁分開顯示？

- **正確**：同一欄「規格」內用**半形逗號**分隔多個選項，例如：`酷洛米,大耳狗` 或 `凱蒂貓, 美樂地`（有無空格皆可）  
  → 顧客頁會顯示兩個選項：「酷洛米」「大耳狗」供客人選擇。
- **也支援**：全形逗號（，）、頓號（、）、分號（;；）、空格、換行，系統會自動當成分隔。
- **若用分欄**：**第一列（最上方）必須是標題列**，在該列設「規格1」「規格2」「規格3」「規格4」，各欄填一個規格名稱；第二列起才是商品資料。標題可寫「規格1」或「**規格1**」皆可。API 會自動合併成逗號分隔字串（例：規格1＝凱蒂貓、規格2＝美樂地 → 顧客頁顯示兩個選項）。
- **常見錯誤**：
  - 標題列寫成「規 格」或「規格 」多空格，系統會讀不到，請用「規格」。
  - 只寫「凱蒂貓 美樂地」中間沒逗號時，系統會依空格拆分；若仍只顯示一個，請改為「凱蒂貓, 美樂地」。

## 七、顧客頁「點規格換圖」說明

- 試算表 **規格** 欄：多個規格用逗號分隔，例：`酷洛米, 凱蒂貓, 大耳狗`。
- 試算表 **規格圖片** 欄：多張圖片 URL 用逗號分隔，**順序要與規格對應**：
  - 第 1 個 URL → 點選「酷洛米」時顯示
  - 第 2 個 URL → 點選「凱蒂貓」時顯示
  - 第 3 個 URL → 點選「大耳狗」時顯示
- 若某規格沒有對應圖片，可留空或少填，顧客頁會改顯示 **商品主圖**（主圖）。
- 後台「新增／編輯商品」時，規格區每列可填「規格名稱」與「該規格圖片 URL」，儲存後會同步寫入試算表 **規格**、**規格圖片** 欄。

## 八、常見問題：商品卡圖片不顯示、規格不正確、現貨/預購沒顯示

- **圖片不顯示**：試算表第一列須有「**商品主圖**」或「**圖片URL**」或「**圖片**」欄位，該列儲存格填**完整圖片網址**（以 https:// 開頭）。若只有規格圖片，顧客頁會用第一張規格圖當商品卡圖。
- **規格只顯示一個**：若用單欄「規格」，請用**半形逗號**分隔（例：`凱蒂貓,美樂,布丁狗`）；若用分欄，第一列必須是「規格1」「規格2」「規格3」標題，第二列起才是資料。修改 Code.gs 後請重新**部署**網頁應用程式。
- **貨況（現貨/預購）在顧客頁沒顯示**：試算表**第一列**須有「**貨況**」（或「現貨預購」「現貨/預購」）欄位，後台儲存時會寫入該欄；顧客頁讀取試算表此欄顯示。請在標題列新增「貨況」，該欄填「現貨」或「預購」，或在後台編輯商品後儲存一次。
